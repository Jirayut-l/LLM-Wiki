# Project Orbit: LLM Wiki Agent

โครงการนี้คือระบบจัดการฐานความรู้ (Knowledge Base) หรือ "Wiki" ที่ควบคุมโดย LLM Agent โดยมีวัตถุประสงค์เพื่อรวบรวมข้อมูลดิบ (Raw Sources) มาสังเคราะห์เป็นความรู้ที่มีโครงสร้าง เชื่อมโยงกัน และพัฒนาไปอย่างต่อเนื่อง

## Project Structure
โครงการนี้ใช้หลักการ **PARA Method** ในการจัดระเบียบข้อมูลภายในโฟลเดอร์ `wiki/`:

- **`raw/`**: เก็บเอกสารต้นฉบับที่ไม่สามารถแก้ไขได้ (Immutable sources) เช่น บทความ, งานวิจัย หรือบันทึกดิบ
- **`wiki/`**: พื้นที่ทำงานของ LLM Agent สำหรับสร้างและจัดการเนื้อหา:
    - **`1. Projects/`**: โครงการระยะสั้นที่มีเป้าหมายและกำหนดเวลาชัดเจน
    - **`2. Areas/`**: พื้นที่ความรับผิดชอบระยะยาวที่ต้องดูแลอย่างต่อเนื่อง (เช่น การเรียนรู้ภาษา Go)
    - **`3. Resources/`**: แหล่งรวบรวมแนวคิด (Concepts), ข้อมูลบุคคล/องค์กร (Entities) และสรุปเนื้อหา (Summaries)
    - **`4. Archives/`**: เก็บสิ่งที่เสร็จสิ้นแล้วหรือไม่ได้ใช้งานจาก 3 หมวดหมู่ข้างต้น
- **`wiki/index.md`**: สารบัญหลักที่รวบรวมหน้าทั้งหมดใน Wiki
- **`wiki/log.md`**: บันทึกลำดับเหตุการณ์การเปลี่ยนแปลงและการทำงานของ Agent

## Core Workflows
1. **Ingest (การนำเข้าข้อมูล):** เมื่อมีการเพิ่มไฟล์ใน `raw/` Agent จะวิเคราะห์เนื้อหา สังเคราะห์ความรู้ และอัปเดตหน้า Wiki ที่เกี่ยวข้อง พร้อมสร้างการเชื่อมโยงแบบ Cross-link
2. **Query (การสอบถาม):** สามารถสอบถามข้อมูลจากฐานความรู้ที่มีอยู่ Agent จะค้นหาจาก `index.md` และสรุปคำตอบพร้อมอ้างอิงแหล่งที่มา
3. **Lint (การดูแลรักษา):** ตรวจสอบความสมบูรณ์ของข้อมูล เช่น ลิงก์เสีย ข้อมูลที่ขัดแย้งกัน หรือหน้าที่ไม่มีการเชื่อมโยง (Orphan pages)

## Conventions
- การเชื่อมโยงเนื้อหาจะใช้รูปแบบ `[[Page Name]]` (Obsidian Style)
- ทุกหน้าใน Wiki จะมี YAML Frontmatter สำหรับเก็บ Metadata และ Tags
- ยึดถือโครงสร้าง PARA และคำสั่งใน `GEMINI.md` เป็นหลัก
