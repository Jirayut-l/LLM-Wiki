# สรุปเทคนิค Prompt Engineering v7

เอกสารฉบับนี้สรุปเทคนิคการออกแบบพรอมต์สำหรับโมเดลภาษาขนาดใหญ่ (LLM) โดยอ้างอิงจาก whitepaper "Prompt Engineering" โดย Lee Boonstra (กุมภาพันธ์ 2025)

## ⚙️ การตั้งค่าเอาต์พุตของ LLM
การออกแบบพรอมต์ที่มีประสิทธิภาพต้องอาศัยการปรับจูนการตั้งค่า (Configuration) ของโมเดลด้วย:

- **Output Length**: จำนวน Token สูงสุดในคำตอบ การจำกัดความยาวไม่ได้ทำให้โมเดลสรุปใจความได้ดีขึ้น แต่เพียงแค่ทำให้โมเดลหยุดตอบเมื่อถึงขีดจำกัด
- **Temperature**: ควบคุมความสุ่ม (Randomness)
  - ค่าต่ำ (ใกล้ 0): คำตอบจะแน่นอนและเป็นข้อเท็จจริงมากขึ้น (เหมาะกับคณิตศาสตร์หรืองานสกัดข้อมูล)
  - ค่าสูง (ใกล้ 1): คำตอบจะมีความหลากหลายและสร้างสรรค์มากขึ้น
- **Top-K & Top-P**: วิธีการสุ่มเลือก Token ที่มีโอกาสเกิดขึ้นสูงสุด

> [!tip] คำแนะนำเบื้องต้น
> สำหรับผลลัพธ์ที่สมเหตุสมผลและมีความคิดสร้างสรรค์พอประมาณ ลองเริ่มต้นด้วย: **Temperature: 0.2, Top-P: 0.95, Top-K: 30**

## 🛠️ กรณีใช้งานจริง (Practical Use Cases)
ประยุกต์ใช้เทคนิคการพรอมต์กับงานเฉพาะด้าน:
- [[prompting-for-programmers|การเขียนพรอมต์สำหรับโปรแกรมเมอร์]]: การเขียนโค้ด, แก้ไขบัค และอธิบายโค้ด
- [[prompting-for-book-summaries|การเขียนพรอมต์เพื่อสรุปหนังสือ]]: การสกัดแนวคิดหลัก และการวิเคราะห์เนื้อหา
- [[prompting-for-learning|การเขียนพรอมต์เพื่อการเรียนรู้]]: เทคนิคไฟน์แมน และการสร้างแผนการเรียนส่วนบุคคล

## ✅ แนวทางปฏิบัติที่ดีที่สุด (Best Practices)
- **ออกแบบด้วยความเรียบง่าย**: พรอมต์ควรสั้น ชัดเจน และเข้าใจง่าย
- **ใช้คำกริยาที่บอกการกระทำ**: เช่น วิเคราะห์ (Analyze), สรุป (Summarize), แปล (Translate)
- **ระบุรูปแบบเอาต์พุตให้ชัดเจน**: การขอให้ตอบเป็น JSON จะช่วยลดการหลอนของโมเดล (Hallucinations)
- **ใช้คำแนะนำ (Instructions) มากกว่าข้อห้าม (Constraints)**: โมเดลตอบสนองต่อการบอกว่า "ให้ทำอะไร" ได้ดีกว่าการบอกว่า "ห้ามทำอะไร"
- **ใช้ตัวแปรในพรอมต์**: เพื่อให้พรอมต์สามารถนำกลับมาใช้ใหม่ได้ง่ายขึ้น
- **JSON Repair**: หากเอาต์พุต JSON ถูกตัดขาดเนื่องจาก Token Limit แนะนำให้ใช้ไลบรารีอย่าง `json-repair` ช่วยแก้ไข
