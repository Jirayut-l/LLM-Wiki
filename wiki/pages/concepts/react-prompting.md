# การพรอมต์แบบ ReAct (Reason & Act)

ReAct เป็นกระบวนทัศน์ (Paradigm) ที่ช่วยให้ LLM สามารถแก้ปัญหาที่ซับซ้อนได้โดยการรวม **การใช้เหตุผลด้วยภาษาธรรมชาติ (Reasoning)** เข้ากับการ **ดำเนินการผ่านเครื่องมือภายนอก (Acting)** เช่น การค้นหาข้อมูล (Search) หรือการรันโค้ด (Code Interpreter)

## การทำงานของ ReAct
ReAct เลียนแบบวิธีการทำงานของมนุษย์ในโลกแห่งความเป็นจริง โดยเราจะใช้เหตุผลด้วยคำพูดและดำเนินการเพื่อหาข้อมูลเพิ่มเติม
1. **คิด (Reason)**: โมเดลวิเคราะห์ปัญหาและสร้างแผนการดำเนินการ
2. **ทำ (Act)**: ดำเนินการตามแผน (เช่น ค้นหา Google)
3. **สังเกต (Observe)**: ดูผลลัพธ์ที่ได้จากการดำเนินการ
4. **วนลูป**: ใช้ผลที่สังเกตได้มาปรับการใช้เหตุผลและสร้างแผนใหม่ จนกว่าจะพบวิธีแก้ปัญหา

> [!info] ก้าวแรกสู่การเป็นเอเจนท์ (Agent)
> ReAct ถือเป็นก้าวแรกที่สำคัญในการพัฒนาโมเดลภาษาให้เป็น Agent ที่สามารถโต้ตอบกับโลกภายนอกผ่าน API ต่างๆ ได้

## ตัวอย่างการใช้งาน
การหาคำตอบที่ต้องใช้ข้อมูลหลายขั้นตอน เช่น "สมาชิกวง Metallica มีลูกรวมกันกี่คน?"
- **Reason**: ต้องหาชื่อสมาชิกแต่ละคนก่อน
- **Act**: ค้นหาชื่อสมาชิกวง Metallica
- **Observe**: พบรายชื่อสมาชิก (James Hetfield, Lars Ulrich, ฯลฯ)
- **Reason**: ต้องหาจำนวนลูกของสมาชิกทีละคน
- **Act**: ค้นหาจำนวนลูกของ James Hetfield... (ทำซ้ำจนครบ)
- **Final Answer**: สรุปยอดรวมลูกทั้งหมด
