# Concept - Go Composite Data Types (ประเภทข้อมูลแบบผสมในภาษา Go)

ประเภทข้อมูลแบบผสม (Composite Types) คือการนำประเภทข้อมูลพื้นฐานมาประกอบกันเพื่อจัดเก็บข้อมูลที่ซับซ้อนมากขึ้น ในภาษา Go จะมี 4 ประเภทหลักที่สำคัญ ได้แก่ Arrays, Slices, Maps และ Structs.

## 1. Arrays (แถวลำดับ)
Array คือชุดของข้อมูลประเภทเดียวกันที่มี **ขนาดคงที่** (Fixed Length) ขนาดของ Array ถือเป็นส่วนหนึ่งของชนิดข้อมูล

**จุดเด่น:**
- เก็บข้อมูลชนิดเดียวกัน
- ต้องระบุขนาดตั้งแต่ตอนประกาศ และเปลี่ยนขนาดไม่ได้

```go
package main

import "fmt"

func main() {
    // ประกาศ Array ขนาด 3 เก็บชนิดข้อมูล string
    var colors [3]string
    colors[0] = "Red"
    colors[1] = "Green"
    colors[2] = "Blue"

    // ประกาศพร้อมกำหนดค่า (Array Literal)
    numbers := [5]int{10, 20, 30, 40, 50}

    fmt.Println("Colors:", colors)
    fmt.Println("First number:", numbers[0])
    fmt.Println("Length of numbers:", len(numbers))
}
```

## 2. Slices (ชิ้นส่วนข้อมูล)
Slice คือโครงสร้างที่ทำงานครอบ Array อีกที แต่มี **ความยืดหยุ่นสูงกว่า** สามารถเพิ่มหรือลดขนาดได้ (Dynamic Size) เป็นประเภทข้อมูลที่ใช้บ่อยที่สุดใน Go

**จุดเด่น:**
- ไม่ต้องระบุขนาดตอนประกาศ
- ใช้ฟังก์ชัน `append()` เพื่อเพิ่มข้อมูลได้

```go
package main

import "fmt"

func main() {
    // ประกาศ Slice
    fruits := []string{"Apple", "Banana"}
    
    // เพิ่มข้อมูลด้วย append
    fruits = append(fruits, "Orange", "Grape")

    // การตัดช่วง (Slicing)
    someFruits := fruits[1:3] // เอา index ที่ 1 ถึงก่อน 3 (คือ 1 และ 2)

    fmt.Println("Fruits:", fruits)
    fmt.Println("Some fruits:", someFruits)
}
```

## 3. Maps (ตารางจับคู่)
Map คือการเก็บข้อมูลแบบ **Key-Value Pair** (คล้ายกับ Dictionary ใน Python หรือ Object ใน JS) โดยที่ Key ต้องมีชนิดข้อมูลที่สามารถเปรียบเทียบกันได้

**จุดเด่น:**
- ค้นหาข้อมูลได้รวดเร็วด้วย Key
- ต้องใช้ฟังก์ชัน `make()` หรือ map literal ในการสร้างก่อนใช้งาน

```go
package main

import "fmt"

func main() {
    // สร้าง Map
    scores := map[string]int{
        "Alice": 90,
        "Bob":   85,
    }

    fmt.Println("Alice's score:", scores["Alice"])
}
```

## 4. Structs (โครงสร้างข้อมูล)
Struct คือการนำข้อมูล **ต่างชนิดกัน** มากลุ่มรวมกันเป็นโครงสร้างเดียว (คล้ายกับ Class ในภาษา OOP แต่ไม่มีการสืบทอด)

**จุดเด่น:**
- นิยมใช้แทน "วัตถุ" หรือ "โมเดลข้อมูล" ในโปรแกรม
- สามารถนิยาม Method ให้กับ Struct ได้

```go
package main

import "fmt"

type User struct {
    ID    int
    Name  string
}

func main() {
    u1 := User{ID: 1, Name: "Somchai"}
    fmt.Printf("User: %+v\n", u1)
}
```

## ค่าเริ่มต้น (Zero Values) ของประเภทข้อมูลแบบผสม

ในภาษา Go เมื่อเราประกาศตัวแปรโดยไม่กำหนดค่าเริ่มต้น จะมี "ค่าเริ่มต้น" (Zero Value) ดังนี้:

1.  **Arrays:** สมาชิกทุกตัวใน Array จะได้รับค่า Zero Value ของชนิดข้อมูลนั้นๆ 
    - เช่น `[3]int` จะได้ `[0 0 0]`, `[2]bool` จะได้ `[false false]`
2.  **Slices:** จะมีค่าเป็น `nil` (ความหมายคือไม่มีข้อมูลและไม่มีการจองพื้นที่)
3.  **Maps:** จะมีค่าเป็น `nil` **ข้อควรระวัง:** คุณไม่สามารถเพิ่มข้อมูลลงใน `nil map` ได้ (จะเกิด error) ต้องใช้ `make()` หรือ map literal ก่อน
4.  **Structs:** ฟิลด์ทุกตัวใน Struct จะได้รับค่า Zero Value ของชนิดข้อมูลนั้นๆ ทันที

## สรุป: การเลือกใช้งาน (How to Choose)

การเลือกใช้ประเภทข้อมูลให้เหมาะสมกับงาน จะช่วยให้โปรแกรมอ่านง่ายและมีประสิทธิภาพ:

-   **ใช้ Array เมื่อ:**
    - คุณรู้จำนวนข้อมูลที่แน่นอนและ **ไม่เปลี่ยนแปลง** (เช่น ชื่อเดือน 12 เดือน, ทิศทั้ง 4)
    - ต้องการความเร็วสูงสุดในการประมวลผล (แต่ปกติใน Go จะใช้ Slice มากกว่า)
-   **ใช้ Slice เมื่อ:**
    - ต้องการเก็บ **รายการข้อมูล** (List) ที่สามารถเพิ่มหรือลดจำนวนได้ตลอดเวลา
    - **เป็นตัวเลือกหลัก (Default)** สำหรับการเก็บชุดข้อมูลใน Go
-   **ใช้ Map เมื่อ:**
    - ข้อมูลมีความสัมพันธ์แบบ **คู่ (Pair)** หรือต้องการค้นหาข้อมูลด้วยชื่อ (Key) แทนการใช้ลำดับ (Index)
    - เช่น เก็บราคาสินค้าโดยใช้ชื่อสินค้าเป็น Key
-   **ใช้ Struct เมื่อ:**
    - ต้องการรวมข้อมูลที่ **เกี่ยวข้องกันแต่คนละประเภท** เข้าด้วยกัน (เช่น ข้อมูลของสมาชิกคนหนึ่งประกอบด้วย ชื่อ, อายุ, อีเมล)
    - ต้องการสร้างโมเดลข้อมูลที่มีพฤติกรรม (Methods) เฉพาะตัว
