# โครงการ LLM Wiki (LLM Wiki Project)

โครงการนี้คือระบบจัดการความรู้ที่ขับเคลื่อนด้วย AI (LLM-driven knowledge base) ที่ออกแบบมาเพื่อเปลี่ยนข้อมูลดิบให้เป็นเครือข่ายความรู้ที่มีโครงสร้างชัดเจนและสืบค้นได้ง่าย โดยใช้มาตรฐานของ Obsidian ในการจัดเก็บข้อมูล

## สถาปัตยกรรมของระบบ (Project Architecture)

โครงสร้างของโครงการแบ่งออกเป็น 3 ส่วนหลัก:

1.  **แหล่งข้อมูลดิบ (Raw Sources - `/raw`)**: พื้นที่สำหรับจัดเก็บเอกสารต้นฉบับที่นำเข้ามา ระบบจะอ่านข้อมูลจากที่นี่แต่จะไม่แก้ไขไฟล์เหล่านี้
2.  **วิกิ (The Wiki - `/wiki`)**: ส่วนประกอบหลักของฐานความรู้ที่สร้างและจัดการโดย AI
    -   `wiki/index.md`: สารบัญที่เน้นเนื้อหาหลักของโครงการ
    -   `wiki/log.md`: บันทึกประวัติการทำงานและกิจกรรมต่าง ๆ ตามลำดับเวลา
    -   `wiki/pages/`: โฟลเดอร์เก็บหน้าความรู้ แยกตามหมวดหมู่ เช่น ทักษะ (Skills), แนวคิด (Concepts), บุคคล/องค์กร (Entities) และบทสรุป (Summaries)
3.  **ทรัพยากร (Assets - `/wiki/assets`)**: สำหรับจัดเก็บรูปภาพและไฟล์ประกอบอื่น ๆ

## กระบวนการทำงานหลัก (Core Workflows)

*   **การนำเข้าข้อมูล (Ingest)**: เมื่อมีการเพิ่มไฟล์ใหม่ในโฟลเดอร์ `raw/` ระบบจะทำการวิเคราะห์ ระบุประเด็นสำคัญ และอัปเดตหน้าวิกิที่เกี่ยวข้องโดยอัตโนมัติ
*   **การสอบถามข้อมูล (Query)**: ผู้ใช้สามารถสอบถามความรู้ภายในระบบ โดย AI จะทำการค้นหาและสังเคราะห์คำตอบพร้อมระบุแหล่งอ้างอิงจากวิกิ
*   **การตรวจสอบความถูกต้อง (Lint)**: ตรวจสอบความสมบูรณ์ของข้อมูล เช่น ลิงก์ที่เสีย หรือเนื้อหาที่ซ้ำซ้อน เพื่อรักษาคุณภาพของฐานความรู้

## มาตรฐานเนื้อหา (Content Conventions)

เพื่อให้ข้อมูลเชื่อมโยงกันอย่างมีประสิทธิภาพ โครงการนี้ใช้มาตรฐานดังนี้:
*   **Obsidian Flavored Markdown (OFM)**: ใช้ `[[wikilinks]]` ในการเชื่อมโยงหน้าต่าง ๆ เข้าด้วยกัน
*   **Obsidian Bases**: ใช้ไฟล์ `.base` เพื่อแสดงข้อมูลในรูปแบบตารางหรือการ์ดที่สามารถกรองและจัดเรียงได้
*   **JSON Canvas**: ใช้ไฟล์ `.canvas` สำหรับการสร้างแผนผังความสัมพันธ์ (Relationship Maps) ที่ซับซ้อน
*   **Defuddle**: ใช้สำหรับดึงเนื้อหาที่สะอาดจากหน้าเว็บเพื่อนำมาประมวลผล
